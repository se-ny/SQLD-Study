# SQLD시험

## 1과목 - 데이터 모델링의 이해

- 모델링의 3가지 특징으로 올바르게 묶인 것은?
    - 단순화, 추상화, 명확화 (단.추.명)
    
- 데이터베이스 모델링 단계를 순서대로 나열한 것은?
    - 개념적 모델링 → 논리적 모델링 → 물리적 모델링 (개.논.물)
    
- ERD 작성 순서로 올바른 것은?
    - 엔터티 도출 → 배치 → 관계 설정 → 관계명 → 차수 → 선택사양 (도.배.설.명.차.선)
    
- ANSI/SPARC 스키마 구조에서 내부 스키마 변경 시 영향을 받지 않은 것은?
    - 물리적 독립성 - 내부 스키마(저장 구조)가 변경되어도 개념 스키마와 외부 스키마 모두 영향을 받지 않는다
    
- 엔터티의 특징으로 옳지 않은 것은?
    - 인스턴스가 1개 이상이면 된다 X
    엔터티는 인스턴스가 2개 이상인 집합이어야 한다. 1개만 있으면 엔터티로 인정되지 않는다
    - 유일한 식별자를 보유해야 한다    O
    2개 이상의 속성을 보유해야 한다 O
    다른 엔터티와의 관계가 필수다    O
    
- 발생 시점에 따른 엔터티 분류에서 두 개 이상의 엔터티를 상속받아 생성되며 수정사항이 많은 것은?
    - 행위 엔터티는 두 개 이상의 엔터티를 상속받아 생성되는 행위로 수강내역, 주문내역이 해당한다
    
- 속성의 분류 중 변형되어 만들어진 속성으로 합계, 평균 등 계산된 값이 해당하는 것은?
    - 파생 속성은 다른 속성으로부터 변형되어 만들어진 속성으로, 합계, 평균, 나이(생년월일로부터 계산) 등이 해당함.

- 여러 개의 하위 속성으로 나눌 수 있는 속성으로 ‘주소’를 예로 들 수 있는 것은?
    - **복합 속성**은 여러 개의 하위 속성으로 나눌 수 있습니다. 
    → 시/도, 도로명, 건물번호로 분리 가능

- ERD에서 존재적 관계와 행위적 관계를 구분하지 않지만, 이를 구분하는 표기법은?
    - ERD (IE, Barker 표기법)에서는 존재적&행위적 관계를 구분하지 않으나 UML에서는 구분한다

- M:N 관계를 관계형 DB에서 표현하기 위해 생성하는 엔터티는?
    - 관계형 DB는 M:N 관계를 직접 표현할 수 없으므로, 교차 엔터티(중간 테이블)를 생성하는 두 개의 1:M관계로 변환한다

- 주식별자의 특성이 아닌 것은?
    - 주식별자는 대표성, 유일성, 최소성, 불변성, 존재성(Not Null)을 만족해야 한다. 가변성은 주식별자의 특성이 아니다

- 식별자 관계(강한 연결)와 비식별자 관계(약한 연결)에 대한 설명으로 옳은 것은?
    - 식별자 관계는 부모-자식 간 상화작용이 의무이다. 비식별자 관계는 선택사항이며, 서로 다른 생명주기를 가진다

- 제2정규화의 내용으로 올바른 것은?
    - 제2정규화(2NF) - 부분 함수 종속 제거. ‘두부이결다조’(도부이결다조)에서 ‘부’가 2NF이다

- 다음 이상현상 중 데이터 갱신시 일부만 갱신되어 모순이 발생하는 것은?
    - 갱신이상은 일부 데이터만 갱신되어 데이터 간 모순이 발생하는 현상이다. 정규화로 이를 방지한다

- 트랜잭션의 ACID 특성 중 트랜잭션끼리 서로 간섭하면 안된다는 것은?
    - 고립성(Isolation)은 트랜잭션끼리 서로 간섭하면 안 된다는 특성이다. 격리 수준에 따라 Dirty Read등의 문제가 발생할 수 있다

- Barker 표기법에서 NULL 허용을 나타내는 기호는?
    - Barker 표기법에서 NULL허용은 ‘o’, NULL 비허용은 ‘*’로 표시한다

- 유일성은 만족하지만 최소성은 만족시키지 못하는 키는?
    - 슈퍼키(Super Key)는 유일성은 만족하지만 최소성을 만족시키지 못하는 키이다. 후보키는 유일성과 최소성을 모두 만족한다.

- 인조식별자의 단점으로 옳은 것은?
    - 인조식별자의 단점은 데이터 중복 가능성과 불필요한 인덱스 생성이다. 장점은 유지보수 용이, 비즈니스 무관 설계이다

- 논리적 모델링의 특징으로 옳지 않은 것은?
    - ERD 작성은 개념적 모델링 단계에서 수행한다. 논리적 모델링은 정규화를 활용하고 재사용성이 높다.

- 관계의 구성 요소 3가지를 바르게 묶은 것은?
    - 관계의 구성은 관계명(Membership), 차수(Cardinality: 1:1, 1:M, M,N), 선택사양(Optionality)이다. “관.차.선”으로 암기

## 2과목 - SQL 기본 및 활용

- SQL명령어 분류에서 GRANT, REVOKE가 속하는 것은?
    - DCL(데이터 제어어)은 권한 관리 명령어로 GRANT(권한 부여)와 REVOKE(권한 회수)가 해당한다

- NULL 관련 함수 중 두 컬럼값이 같으면 NULL, 다르면 첫 번째 값을 반환하는 함수는?
    - NULLIF(컬럼1, 컬럼2) - 두 값이 같으면 NULL, 다르면 첫 번째 값을 반환한다

- ROUND(15.58, -1)의 결과값은?
    - ROUND에서 자릿수가 음수이면 정수 부분을 반올림한다. -1은 일의 자리에서 반올림 →20

- LIKE연산자에서 정확히 1개의 문자를 의미하는 와일드카드는?
    - %: 0개 이상의 임의 문자 / _: 정확히 1개의 문자를 의미

- 집계함수를 WHERE절에서 사용할 수 없을 때 대신 사용하는 절은?
    - WHERE절에서는 집계함수 사용이 불가하나, HAVING 절에서는 집계함수를 사용할 수 있다. HACING은 그룹 단위 조건이다.

- Oracle에서 ORDER BY ASC(오름차순) 시 NULL은 어떻게 처리되는가?
    - Oracle은 NULL을 최댓값으로 처리하므로 ASC 정렬 시 맨 뒤에 위차한다. SQL Server는 NULL을 최솟값으로 처리한다.

- JOIN만 단독으로 사용 시 의미하는 조인 종류는?
    - JOIN 단독 사용 시 INNER JOIN을 의미하며, 두 테이블의 동일한 값이 있는 행만 반환한다. NULL은 조회대상이 아니다.

- 두 테이블의 모든 행을 조합하는 Cartesian Product 연산에 해당하는 조인은?
    - CROSS JOIN은 두 테이블의 모든 행을 조합해서 출력한다.
    행 수 = 테이블A행 수 x 테이블B 행 수

- 두 SELECT결과의 합집합에서 중복을 제거하고 내부 정렬도 수행하는 연산자는?
    - UNION은 중복을 제거하고 내부 정렬을 수행한다. UNION ALL은 중복을 허용한다. 집합 연산자 사용시 두 집합의 스키마가 일치해야 한다.

- Oracle에서 상위 5개 데이터를 조회할 때 ROWNUM 사용의 유의사항으로 옳은 것은?
    - ROWNUM은 조회 결과에 순차 번호를 부여하는 가상 컬럼으로, ORDER BY 이전에 부여된다. 따라서 서브쿼리에서 정렬 후 ROWNUM 으로 필토링해야 한다.

- 계층형 질의에서 PRIOR자식=부모 방향은?
    - PRIOR 자식=부모: 순방향(부모→자식), PRIOR 부모=자시ㄱ: 역방향(자식→부모)

- DDL의 TRUNCATE와 DML의 DELETE의 차이로 옳은 것은?
    - TRUNCATE는 기억장에서 바로 데이터가 지워지므로
