# ✍️ SQLD 필기 시험 완벽 대비 모의고사 

## 📌 Q1. 오답
* **문제:** SQL 명령어 중 테이블의 구조 자체를 변경하거나 생성, 삭제하는 DDL(Data Definition Language)에 해당하지 않는 것은 무엇인가요?
  1. DELETE
  2. ALTER
  3. CREATE
  4. DROP

<details>
<summary>💡 정답 및 해설 보기 (클릭)</summary>

* **정답:** 1번
* **해설:** DELETE는 테이블의 구조는 남겨둔 채 내부의 데이터(행)를 삭제하는 DML(Data Manipulation Language)입니다.

</details>

* **문제:** 관계형 데이터베이스에서 NULL 값의 특성에 대한 설명으로 올바른 것은 무엇인가요?
  1. 공백 문자(Space) 혹은 빈 문자열(")과 완전히 같다.
  2. NULL 연산(예: NULL+10)의 결과는 언제나 NULL이다.
  3. 조건절에서 NULL울 찾을 때는 기호 '='를 사용한다.
  4. 숫자 0(Zero)과 동일한 의미를 가진다.

<details>
<summary>💡 정답 및 해설 보기 (클릭)</summary>

* **정답:** 2번
* **해설:** 알 수 없는 값(NULL)에 어떤 수를 더하거나 연산해도 그 결과는 여전히 알 수 없는 값(NULL)이 됩니다.

</details>

* **문제:** 다음 중 GROUP BY 절을 사용할 때, 집계된 결과에 대한 필터링(조건)을 적용하기 위해 사용하는 전용 절은 무엇인가요?
  1. ORDER BY
  2. HAVING
  3. WHERE
  4. SELECT

<details>
<summary>💡 정답 및 해설 보기 (클릭)</summary>

* **정답:** 2번
* **해설:** HAVING절은 GROUP BY에 의해 그룹화된 결과 세트에 대해 조건을 부여할 때 사용합니다.

</details>

* **문제:** 트랜잭션(Transaction)의 특성 중 '트랜잭션의 연산은 데이터베이스에 모두 반영되든지 아니면 전혀 반영되지 않아야 한다(All or Nothing)'는 특성은 무엇인가요?
  1. 원자성 (Atomicity)
  2. 일관성 (Consistency)
  3. 고립성 (Isolation)
  4. 영속성 (Durability)

<details>
<summary>💡 정답 및 해설 보기 (클릭)</summary>

* **정답:** 1번
* **해설:** 원자성은 트랜잭션 내의 모든 명령이 완벽히 수행되거나, 하나라도 실패하면 모두 취고(Rollback)되어야 함을 의미합니다.

</details>

* **문제:** 여기서부

  2. 반드시 해당 업무에서 필요로 하고 관리하고자 하는 정보여야 한다.

  4. 유일한 식별자에 의해 식별이 가능해야 한다.

<details>
<summary>💡 정답 및 해설 보기 (클릭)</summary>

* **정답:** 3번
* **해설:** 엔터타는 영속적으로 존재하는 두 개 이상의 인스턴스 집합이어야 하므로, 단 하나의 인스턴스만 가지는 엔터니는 부적절합니다.
